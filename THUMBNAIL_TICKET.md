# Engineering Ticket — YouTube Thumbnail Automation Pipeline

**Project:** Encoded.Ai — YouTube Thumbnail System
**Related ticket:** PIPELINE_TICKET.md (Carousel Automation)
**Priority:** Medium
**Type:** Feature — New Module (can be standalone or integrated into Carousel Studio)

---

## Background

Encoded.Ai produces regular YouTube video releases. Every video needs a thumbnail.
The current process is partially manual and involves AI image generation, layout
composition, text generation, and branding application. This ticket covers automating
that end-to-end flow and integrating it into the same production infrastructure
being built for the carousel pipeline.

This can be built as a separate page within the existing Carousel Studio app
(recommended — shared infrastructure, shared design system) or as a fully
standalone project. The decision is left to the developer, but the shared approach
is preferred to avoid duplicating the render service, storage, and notification
systems described in the carousel ticket.

---

## Current Manual Flow

```
1. Image is generated via Nano Banana using a structured prompt
   based on the video content

2. Generated image is placed on the RIGHT side of the layout,
   centered vertically

3. Text block is placed on the LEFT side, derived from video content,
   following predefined typography and layout rules

4. Styling rules applied: font, size, colors, layout behavior

5. Encoded.Ai logo placed in the top-left corner

6. Final composite exported as a 1280×720 PNG (YouTube standard)
```

---

## Target Layout — Thumbnail Anatomy

```
┌─────────────────────────────────────────────────────────┐
│ [LOGO]                                                   │
│                                                          │
│   HEADLINE TEXT          │   AI GENERATED IMAGE         │
│   bold, large            │   right side, vertically     │
│                          │   centered, full height       │
│   Supporting line        │                              │
│   smaller, muted         │                              │
│                          │                              │
└─────────────────────────────────────────────────────────┘
  1280px wide × 720px tall
```

The left side occupies approximately 50–55% of the canvas width.
The right side image occupies the remaining 45–50%, full height, with a subtle
fade/gradient on the left edge where it meets the text column.

## What Needs to Be Built

### Module 1 — Image Generation Integration

**Goal:** Accept a video title and description, generate a relevant image via Nano Banana API.

1. Accept structured input:
```json
{
  "video_title": "ENCODED Member Story: Matt Wagner | 3 Weeks of Subconscious Programming",
  "video_description": "Short description of video content",
  "style_notes": "Optional — e.g. 'dark, cinematic, human subject'"
}
```

2. Construct a structured prompt following Nano Banana's requirements
3. Call Nano Banana API, receive generated image
4. Store image temporarily for layout composition step
5. Return image URL or base64 for use in the next step

**Prompt structure for Nano Banana** (to be refined with the content team):
- Subject: relevant to video topic
- Style: dark, editorial, cinematic — consistent with Encoded brand
- Composition: subject should sit naturally on the right half of a wide frame
- No text in the generated image — text is added in layout step
- Aspect hint: wide crop, 16:9 friendly

---

### Module 2 — Text Generation

**Goal:** Generate the headline and supporting line from video metadata.

Input: video title + description
Output: two text fields:
- `headline` — bold, large, punchy. Usually a restatement of the core claim.
  Max 6–8 words. Should work visually at large size on the left side.
- `subline` — smaller supporting text. 1 line. Context or name.
  e.g. "Matt Wagner — 3 Weeks In" or "The Frequency Era"

This can be generated by Claude API or written manually.

Suggested Claude prompt approach:
```
Given this YouTube video title and description, generate:
1. A short thumbnail headline (max 7 words, bold claim, no punctuation)
2. A one-line subtext (name, context, or series — max 6 words)

Title: {title}
Description: {description}

Respond only with JSON: { "headline": "...", "subline": "..." }
```

---

### Module 3 — Layout Renderer

**Goal:** Compose the final thumbnail as a 1280×720 PNG.

**Approach:** Build as an HTML template rendered by the same Playwright render service
used for carousels (from the carousel pipeline ticket). This keeps infrastructure unified.

**HTML template structure:**
```html
<body style="width:1280px; height:720px; overflow:hidden; background:#111009; position:relative;">

  <!-- Background: full canvas dark -->

  <!-- Right side: AI image, full height, fading left edge -->
  <div class="thumb-image-col">
    <img src="{generated_image_url}">
    <div class="thumb-image-fade"></div> <!-- gradient overlay on left edge -->
  </div>

  <!-- Left side: text block -->
  <div class="thumb-text-col">
    <div class="thumb-logo">
      <img src="assets/encoded-logo.svg">
    </div>
    <div class="thumb-headline">{headline}</div>
    <div class="thumb-subline">{subline}</div>
  </div>

</body>
```

**Styling rules:**

| Element | Rule |
|---------|------|
| Canvas | 1280×720px, background `#111009` |
| Logo | Top-left, ~120px wide, `assets/encoded-logo.svg` |
| Headline | DM Serif Display, 72–88px, color `#EDE8DC`, left-aligned, max 2 lines |
| Subline | DM Sans 300, 28px, color `#68665e`, left-aligned, below headline |
| Image column | Right 50% of canvas, full height, `object-fit: cover`, `object-position: center` |
| Image fade | Linear gradient on the left edge of the image: `rgba(17,16,9,1)` → `rgba(17,16,9,0)` |
| Left padding | 72px from canvas edge |
| Vertical alignment | Text block centered vertically in left column |
| Gold accent | Optional: a 3px vertical gold bar (`#C9A84C`) to the left of the headline |

**The image fade is critical** — it blends the right-side image into the dark
background so the text on the left remains readable at all times regardless of
image content.

---

### Module 4 — Render and Export

The composed HTML template is passed to the same render service used for carousels:

```
POST /render
{
  "html": "<full html string>",
  "width": 1280,
  "height": 720,
  "slides": 1
}
```

Returns a single 1280×720 PNG ready for YouTube upload.

### Module 5 — Studio UI (new tab in Carousel Studio)

Add a "Thumbnails" tab to the existing Carousel Studio app with:

1. **Input form:**
   - Video title (text field)
   - Video description (textarea)
   - Optional: manual headline override
   - Optional: manual subline override
   - Optional: style notes for image generation
   - Optional: manual image upload (skip Nano Banana)

2. **Preview panel:**
   - Live preview of the composed thumbnail at reduced scale
   - Toggle to swap between generated image and manual upload

3. **Generate button:**
   - Triggers image generation → text generation → layout render
   - Shows progress (generating image → composing layout → rendering)

4. **Export:**
   - Download 1280×720 PNG
   - Optional: save to shared storage (same bucket as carousel PNGs)

## Full Automated Flow (target state)

```
Video title + description entered
        ↓
Claude API generates headline + subline
        ↓
Nano Banana generates background image from structured prompt
        ↓
HTML template is assembled with image, text, logo
        ↓
Render service (Playwright) screenshots at 1280×720
        ↓
PNG saved to storage
        ↓
Download link returned to UI
        ↓
Human reviews → uploads to YouTube manually
   (YouTube API automation is out of scope for v1)
```

## Design System Alignment

Thumbnails share the same design tokens as carousels.
The same `styles.css` color variables apply:

```css
--bg:    #111009;   /* canvas background */
--white: #EDE8DC;   /* headline text */
--gray:  #68665e;   /* subline text */
--gold:  #C9A84C;   /* optional accent */
--fh:    'DM Serif Display', serif;
--fb:    'DM Sans', sans-serif;
```

This ensures visual consistency between thumbnails, carousels, and the website.

---

## Branding Rules

- Logo: top-left corner, Encoded.Ai mark (SVG preferred for sharpness)
- Brand name never appears as text on thumbnails (logo only)
- Gold accent used at most once per thumbnail
- No cluttered layouts — maximum two text elements (headline + subline)
- Dark background always — never white or light thumbnails

---

## Out of Scope for v1

- Automatic YouTube upload (requires OAuth per channel — add in v2)
- A/B thumbnail testing
- Animated thumbnails (GIF format — architecture note: render service
  can support this in future with the same video capture approach
  described in the carousel animation phase)
- Multiple thumbnail variants per video
